この問題を「巡回セールスマン問題」と呼ぶ最初の出版物は、1949 年の Julia Robinson による RAND Corporation レポート「On the Hamiltonian game (a traveling salesman problem)」で見つかります。

彼女は問題を次のように定義しています：

「この注記の目的は、巡回セールスマン問題に関連する問題を解くための方法を提供することです。元の問題の説明を与える価値があるように思われます。1 つの定式化は、ワシントンから始まり、すべての州都を訪問してからワシントンに戻るセールスマンの最短経路を見つけることです。

より一般的には、平面内の n 個の与えられたポイントを含む最短 CLOSED CURVE を見つけることです。」

例えば、以下の都市のセットの場合：
0, 0
1, 0
2, 0
0, 1
1, 1
2, 1
1, 2
2, 2
これは以下のように表示できます：
+ + +
+ + +
  + +
最短パスは：
 _____
|__   |
   |__|

したがって、このパスの長さを出力する必要があります：
8.00

標準入力から「%f, %f\n」の形式で都市座標のセットを読み込み、すべてのこれらの都市を含む最短可能なパスの長さを「%.2f」の形式で出力するプログラムを書いてください。

プログラムは最大 11 個の都市でテストされます。

このディレクトリには、この演習のすべての退屈な部分を含むファイル tsp.c とプログラムをテストするのに役立つ例ファイルが見つかります。

ヒント：sqrtf を使用するには、コンパイルコマンドの最後に -lm を追加してください。

例えば、これは動作するはずです：
$> cat square.txt
1, 1
0, 1
1, 0
0, 0
$> ./tsp < square.txt | cat -e
4.00$
